## 6. Implementation Structure

### Recommended File Structure

```
src/
  content/
    imageManifest.ts    # All 26 images with metadata
  data/
    services.ts        # Update to use local images
```

### Manifest Fields (per image)
- `id`, `filename`, `alt`, `category`, `tags`
- `orientation` (landscape | portrait | square)
- `featured` (boolean)
- `heroCandidate` (boolean)
- `recommendedPlacement` (string[])
- `priority` (1–5)
- `serviceAssociations` (string[])
- `projectGroup` (string | null)
- `beforeAfter` ('before' | 'after' | null)
- `notes` (string)
- `cropRecommendation` (string | null)

## 7. Image Optimization Recommendations

### Cropping
- **`2.jpg`** — Crop bottom 20–25% to remove dirt and generator
- **`4.jpg`** — Crop or retouch electrical boxes and drain grate
- **`6.jpg`** — Retouch or crop out power drill on sill
- **`10.jpg`** — Tight crop on column capital; exclude sawhorse
- **`14.jpg`** — Crop out drink bottle, old walkway, leaves
- **`1.1.jpg`** — Crop out trash bin and bag if possible

### Hero Overlays
- **`7.jpg`**, **`1.jpg`** — Use a subtle dark gradient (bottom or full) for text readability. Test contrast with white/cream text.
- **`4.jpg`** — Darker overlay may be needed; brick can compete with text.

### Format
- Convert all to **WebP** for production; keep JPG originals as fallback
- Use `srcset` for responsive loading (e.g., 640w, 1024w, 1920w for heroes)

### Cohesion & Performance
- **Consistency:** Stick to hero-worthy and featured images for homepage; avoid mixing weak images with strong ones
- **Lazy loading:** Use `loading="lazy"` for below-fold images
- **Aspect ratio:** Reserve space with `aspect-ratio` CSS to prevent layout shift
- **Priority:** Preload `7.jpg` (or chosen hero) for LCP
